Reflection

In this project, we developed an AI agent designed to automate the restaurant reservation process. The user provides preferences such as cuisine type, location, budget, and preferred reservation time. Based on these inputs, the agent identifies a suitable restaurant, selects the best available option, completes the reservation, and finally sends the reservation details to a selected contact via WhatsApp using Twilio.

At this stage of the project, we did not integrate with a real restaurant API. Instead, we created our own data files containing restaurant information, available reservation times, and contact details. These files served as our data source and simulated the behavior of a real API. This approach allowed us to develop, test, and demonstrate the complete workflow of the agent while keeping the system flexible enough to replace the data files with a real API in the future.

One of the main challenges we encountered was designing a workflow that coordinated several sequential tools. Each step depends on the successful completion of the previous one: understanding the user’s request, finding a suitable restaurant, checking availability, making the reservation, and sending the confirmation message through Twilio. Because the agent performs multiple tool calls in sequence, the overall response time was sometimes longer than expected. For example, before sending the WhatsApp message, the agent first searches the contacts CSV file, retrieves the correct phone number, activates the Twilio service, and only then sends the message. To improve the user experience, we added progress updates that inform the user what the agent is doing at each stage. This prevents the application from appearing unresponsive during longer operations.

Throughout the project, we gained hands-on experience in combining a large language model with external tools and data sources to build an autonomous AI agent capable of completing an end-to-end task. We also learned how to integrate Twilio for WhatsApp messaging and gained a better understanding of workflow orchestration, tool integration, and the importance of user experience when building AI-powered applications.

If we continue developing this project, our next step would be to replace the local data files with a real restaurant API that provides live information about restaurants, availability, and reservations. We would also like to add features such as comparing restaurants based on ratings and reviews, suggesting alternative restaurants when no reservations are available, allowing users to modify or cancel reservations, and improving error handling and performance to make the system more robust.

Overall, this project gave us valuable experience in designing and developing an AI agent that integrates a language model, external data sources, and third-party services into a single automated workflow. Beyond the technical implementation, it strengthened our understanding of AI agent architecture, tool integration, workflow design, and the importance of creating a smooth and user-friendly experience.
